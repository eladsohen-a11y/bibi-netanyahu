1. סקירת הפרויקט

מה זה: אפליקציית ווב סאטירית שבה המשתמש שואל שאלות ודמותו המאוירת של בנימין נתניהו ("ביבי") עונה בסגנון הרטורי האותנטי שלו — באמצעות AI (Claude). הדמות משנה הבעות פנים בהתאם לטון, מד לחץ ומדדים שונים מטפסים, יש שאלות "אסורות" שמפעילות מצב חירום, ויש תגובות נדירות.

מטרת המשחק: המשתמש מנסה "להפיל" את ביבי באמצעות שאלות חודרות יותר ויותר — אבל ביבי תותח עם קור רוח, אז זה לוקח מאמץ. זה יוצר התמכרות: "מה יקרה אם אשאל X?".

קהל יעד: ויראלי — תוצאה ניתנת לשיתוף בוואטסאפ.

2. ארכיטקטורה ותשתית טכנית

צד לקוח (דפדפן)

  • HTML / CSS / JavaScript טהורים
  • בלי React/Vue/Build — פשוט וקל לתחזוקה
  • פונט Heebo (Google Fonts)
  • תמיכה מלאה ב-RTL ובמובייל

שרת (Node.js)

  • Express — שרת מינימלי
  • מגיש את האתר + מתווך לקריאות AI
  • שומר על המפתח הסודי
  • אכיפת הגבלות (rate limit + תקרה יומית)

AI

  • Anthropic Claude API
  • SDK רשמי: @anthropic-ai/sdk
  • מודל: claude-sonnet-4-6 (איכותי)
  • Structured Outputs — JSON תקין מובטח

תלויות מינימליות

  • express
  • dotenv
  • express-rate-limit
  • @anthropic-ai/sdk
עיקרון מנחה: בלי overengineering. כל החלטה נבחנה לפי "האם זה תורם לחוויה?". פלאגינים, frameworks ותלויות מיותרות נדחו במכוון.

4. שכבת ה-AI (Claude) — איך זה עובד

הזרימה

  1. המשתמש כותב שאלה ולוחץ "שאל".
  2. הדפדפן שולח POST /api/ask עם השאלה + היסטוריית השיחה.
  3. השרת בונה הודעות, מוסיף System Prompt (האופי של ביבי), שולח ל-Claude.
  4. Claude מחזיר JSON מובנה: { text, mood }.
  5. השרת מחזיר ללקוח; הדפדפן מציג את התשובה ומחליף תמונת ביבי לפי ה-mood.

זיכרון שיחה

הדפדפן שולח את 12 ההודעות האחרונות בכל קריאה — ביבי "זוכר" הקשר, יכול להתייחס למה שנאמר קודם, ואפילו לסתור את עצמו (זהב סאטירי).

JSON מובטח (Structured Outputs)

במקום לסמוך על Claude שיחזיר פורמט נכון, מגדירים סכמה: {text: string, mood: enum}. Claude חייב לציית — אז ההבעה תמיד תקפה ולא תיכשל הפענוח.

Fallback חכם

אם אין מפתח/קרדיט/חיבור — האפליקציה נופלת אוטומטית למאגר תשובות סטטי (responses.js) ולא נשברת. הדפדפן מזהה את מצב השרת דרך GET /api/health.

6. כיול האופי וקור הרוח

נקודה קריטית. ביבי הוא שחקן ותיק עם עצבי ברזל — הוא לא נשבר מהר. שאלה רגישה בודדת לא צריכה להעיף אותו לפאניקה.

שכבת ההוראות (ל-Claude)

ה-System Prompt מורה לו במפורש: שלוט, חייך, רד לעקיצה — אבל אל תיכנס לפאניקה משאלה רגישה רגילה. רק התגרות אמיתית או נושא שבאמת כואב יוצרים תגובה רגשית.

תדירות הבעות פנים

  • רוב הזמן answering · sly · proud · defiant · dodging
  • לפעמים suspicious · accusing
  • נדיר angry · nervous-smile · emotional

8. הפעלת האפליקציה מקומית

  1. פתח את התיקייה C:\Users\pblel\קלוד\bibi-app
  2. ודא שיש מפתח ב-.env (ANTHROPIC_API_KEY=sk-ant-...)
  3. הפעל בטרמינל: node server.js
  4. פתח בדפדפן: http://localhost:3000
  5. אם שינית קבצים — Ctrl+Shift+R לרענון קשה
חשוב: ה-AI עובד רק דרך localhost:3000. פתיחת הקובץ ישירות נופלת אוטומטית למצב סטטי.

9. החלטות עיצוב ומוצר לאורך הדרך

  • HTML/CSS/JS טהורים במקום React — פשטות, אפס build, קל לתחזוקה.
  • Node + Express במקום serverless — שרת אחד מטפל בהכל, פריסה פשוטה.
  • Sonnet ולא Haiku — האיכות והעוקצנות חשובות יותר מהחיסכון בפרויקט סאטירה.
  • Structured Outputs במקום פרסור טקסט — אפס תקלות פורמט.
  • Fallback סטטי תמיד פעיל — אף פעם לא נשבר.
  • שכבת משחק בקוד (לא תמונות נוספות) — מנוצל את 12 הציורים הקיימים ומוסיף אפקטים בקוד.
  • בלי סאונד — בעיות autoplay במובייל, יותר נזק מתועלת.
  • טיקר תחתון הוסר — היה רעש מיותר, פוגע בנקיון העיצוב.
  • כיול קור רוח — שאלה רגישה ≠ פאניקה. רק חפירה אמיתית סודקת.

10. שלבים הבאים — העלאה לאינטרנט

  1. מאגר GitHub: ליצור repo ריק בכתובת github.com/<שם>/bibi-app ולדחוף את הקוד (הקוד כבר מוכן עם git מקומי).
  2. פריסה ב-Render: חשבון חינמי, חיבור ל-GitHub repo. הקובץ render.yaml מגדיר הכל אוטומטית — רק להגדיר את ה-ANTHROPIC_API_KEY בממשק.
  3. דומיין (אופציונלי): חיבור דומיין נחמד דרך Render.
  4. תוספות עתידיות אם נרצה:
    • תמונות חדשות (שומר ראש, דוכן ריק, פאניקה קיצונית) — המערכת מוכנה לקלוט אותן.
    • "כרטיס שיתוף" — תמונה מעוצבת אוטומטית עם השאלה והתשובה.
    • Modes שונים — ריאיון / מסיבת עיתונאים / חקירה.
    • אנליטיקס — אילו שאלות הכי פופולריות.
